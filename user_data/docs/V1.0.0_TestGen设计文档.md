# TestGen — 通用测试用例自动生成系统 设计文档

> 版本: V1.0.0  
> 日期: 2026-07-09

---

## 1. 概述

TestGen 是一个模块化的测试用例自动生成系统，支持从多种输入源自动生成测试用例，并通过 LLM（大型语言模型）增强生成质量。

### 1.1 核心能力

| 能力 | 说明 |
|------|------|
| **多输入源** | OpenAPI/Swagger 规范、Python 源代码 AST 分析、自然语言描述 |
| **多输出格式** | pytest 可执行代码、JSON/YAML 结构化数据、Excel/CSV 表格 |
| **LLM 增强** | 集成 OpenAI 兼容 API，智能生成高质量测试用例 |
| **回退机制** | LLM 不可用时自动切换为规则模板模式 |
| **模块化架构** | 解析器/生成器/输出适配器完全解耦，易于扩展 |

## 3. 核心数据模型

### 3.1 GenerationContext

贯穿整个生成流程的统一上下文，承载：

- `input_source` — 输入来源（OPENAPI / SOURCE_CODE / NATURAL_LANG）
- `output_format` — 输出格式
- `test_types` — 测试类型列表
- `api_endpoints` — 解析后的 API 端点列表
- `functions` / `classes` — 解析后的函数/类列表
- `natural_lang_desc` — 自然语言描述
- LLM 配置（model, temperature, enabled）

### 3.2 关系

```
GenerationContext (1)
  ├── APIEndpoint (N)       — API 端点
  │     ├── Parameter (N)   — 请求参数
  │     └── responses       — 响应定义
  ├── FunctionDef (N)       — 函数/方法
  │     └── Parameter (N)   — 函数参数
  ├── ClassDef (N)          — 类定义
  │     └── FunctionDef (N) — 类方法
  └── TestSuite (N)         — 测试套件（生成结果）
        └── TestCase (N)    — 测试用例
              └── TestStep (N) — 测试步骤
```

## 4. 模块详解

### 4.1 输入解析器 (Parsers)

| 解析器 | 输入 | 核心实现 |
|--------|------|----------|
| `OpenAPIParser` | OpenAPI 3.x / Swagger 2.x (JSON/YAML) | 解析 paths → APIEndpoint 列表，提取参数、请求体、响应 schema |
| `CodeParser` | Python 源文件/目录 | AST 遍历 → `ast.NodeVisitor` 提取 FunctionDef、ClassDef，计算圈复杂度 |
| `NaturalLangParser` | 纯文本/文件 | 启发式检测测试类型关键字，存入 `natural_lang_desc` 供 LLM 使用 |

### 4.2 生成引擎 (Generators)

**双模式设计：**

#### LLM 模式（默认）
```
Input → PromptBuilder → LLMClient.chat_json() → 解析 JSON 响应 → TestSuite[]
```

- `PromptBuilder` 根据解析结果针对不同类型构建专门的 system/user prompt
- `LLMClient` 封装 OpenAI 兼容 API，支持自定义 base_url（Ollama 等本地模型）
- 强制 `response_format: json_object` 确保结构化输出

#### 规则模板模式（`--no-llm` 回退）
```
Input → 预定义规则 → TestSuite[]
```

每种测试类型有独立的回退生成逻辑：
- API: 正常请求 + 参数缺失 + 空请求体
- Unit: 正常调用 + 边界值 + 异常输入（高复杂度函数）

### 4.3 输出适配器 (Output Adapters)

| 适配器 | 输出格式 | 特性 |
|--------|----------|------|
| `PytestAdapter` | `.py` 测试文件 | Jinja2 渲染，生成 `def test_xxx()` 骨架 + conftest.py |
| `JSONAdapter` | `.json` | 结构化 JSON，多彩套件自动合并为 `_all_test_suites.json` |
| `YAMLAdapter` | `.yaml` | 人类可读的 YAML 格式 |
| `ExcelAdapter` | `.xlsx` | 带优先级着色（红/黄/绿）、单元格合并、自动列宽 |
| `CSVAdapter` | `.csv` | UTF-8 BOM 编码，Excel 可直接打开 |

## 5. CLI 使用指南

### 5.1 命令总览

```bash
# 安装
pip install -r requirements.txt

# 查看帮助
python -m testgen --help

# 检查环境
python -m testgen check

# 列出支持的格式
python -m testgen list-formats
```

### 5.2 生成测试用例

```bash
# 从 OpenAPI 文档生成 pytest（无 LLM）
python -m testgen generate -i api.yaml -s openapi -f pytest --no-llm

# 从 Python 代码生成单元测试
python -m testgen generate -i ./src/ -s code -f pytest -t unit

# 从需求文档生成 Excel（使用 LLM）
python -m testgen generate -i requirements.txt -s nl -f excel

# 指定 LLM 模型和输出目录
python -m testgen generate -i api.yaml -s openapi -f json \
    --model gpt-4o --temperature 0.3 -o ./my_output

# 多测试类型
python -m testgen generate -i api.yaml -s openapi -f pytest -t api -t e2e
```

### 5.3 环境变量

```bash
# 必需（LLM 模式）
export OPENAI_API_KEY="sk-xxx"

# 可选：自定义 API 端点（如本地 Ollama）
export OPENAI_BASE_URL="http://localhost:11434/v1"
```

## 6. 扩展指南

### 6.1 添加新输入源

1. 在 `testgen/parsers/` 创建新解析器，继承 `BaseParser`
2. 在 `InputSource` 枚举中添加新值
3. 在 `Orchestrator._parsers` 中注册

### 6.2 添加新输出格式

1. 在 `testgen/outputs/` 创建新适配器，继承 `BaseOutputAdapter`
2. 在 `OutputFormat` 枚举中添加新值
3. 在 `Orchestrator._output_adapters` 中注册

### 6.3 支持新语言

1. 在 `testgen/parsers/` 添加对应语言的 AST 解析器（如 `java_parser.py`）
2. 实现 `BaseParser.can_handle()` 判断文件类型
3. 在 `CodeParser` 或独立解析器中注册

## 7. 核心抽象接口

```python
class BaseParser(ABC):
    def parse(self, context: GenerationContext) -> GenerationContext: ...
    def can_handle(self, context: GenerationContext) -> bool: ...

class BaseGenerator(ABC):
    def generate(self, context: GenerationContext) -> list[TestSuite]: ...

class BaseOutputAdapter(ABC):
    def write(self, suites: list[TestSuite], context: GenerationContext) -> list[str]: ...
    def format_name(self) -> str: ...
```
